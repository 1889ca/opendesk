# OpenDesk QA Issues — 2026-04-09

First real QA pass. Tested: dashboard, document editor, share flow, toolbar features, comment system, keyboard shortcuts, and document creation flows.

---

## 🔴 CRITICAL (security / data loss)

### 1. Share permissions are completely non-functional
**File:** `modules/app/internal/share/`  
The Share dialog lets you set permission to "Viewer", "Commenter", or "Editor" and click "Create link". All three levels redirect to the full, unrestricted editor. Visiting a "Viewer" share link in a fresh tab opens the document with the complete editing toolbar, full write access, and registers the visitor as an active collaborator. The permission model is entirely unimplemented.

### 2. Migration 002_references.sql fails on every startup
**Log:** `[opendesk] schema init failed: Migration 002_references.sql failed: syntax error at or near "references" — continuing anyway`  
The word `references` is a reserved PostgreSQL keyword. Using it unquoted as an identifier fails. The server silently continues, leaving the References feature in an unknown/broken state on every cold start.

### 3. Delete buttons nested inside anchor tags — no stopPropagation
**File:** `modules/app/internal/doc-list/doc-row.ts`  
All 15 Delete buttons on the dashboard are rendered as children of `<a>` elements with `type="submit"` and no click handler to stop propagation. A click on Delete fires both the delete action and the link navigation. Confirmed with JavaScript inspection: `insideAnchor: true`, `hasStopPropagation: "no onclick attr"`. This is a data-loss risk.

### 4. `window.prompt()` used for New Spreadsheet and New Presentation
**File:** `modules/app/internal/doc-list/doc-list.ts:73`  
```ts
const titleText = prompt(`${meta.label} title:`);
```  
Clicking "New Spreadsheet" or "New Presentation" opens a native browser `prompt()` dialog. This blocks all JavaScript execution (including CDP/automation), shows a jarring "localhost says:" system dialog, cannot be styled, and behaves differently across browsers. "New Document" correctly uses a custom modal template picker — the other two should too.

## 🟠 HIGH (broken features)

### 5. Find & Replace shortcut (⌘+H) is listed but completely broken
The keyboard shortcuts help dialog (?) lists "Find and replace → ⌘+H". Pressing ⌘+H does nothing. On macOS, ⌘+H also minimizes the window, making this doubly harmful.

### 6. PDF export button silently fails
Clicking "PDF" in the toolbar highlights the button but triggers nothing — no download starts, no loading indicator appears, no error is shown. The export API endpoint exists (returns 401 when called directly), so the UI action is failing silently.

### 7. Word count shows impossible values after commenting
After opening the comment panel, the status bar displayed **"15 of 13 words selected"** — selected count higher than total. This persisted through multiple toolbar interactions and panel opens/closes before eventually clearing itself.

### 8. Comment anchor spans the entire document instead of selected text
When a comment was added to one line, the yellow comment highlight extended from that line all the way through every subsequent element in the document. The anchor range is not correctly scoped to the selection.

### 9. `[onLoadDocument] invalid input syntax for type uuid: "undefined"`
Server log shows this error, indicating a code path attempts to load a document with UUID `"undefined"` (the string). Likely triggered when the spreadsheet or presentation editor loads without a `?doc=` parameter — the editors apparently don't validate or handle missing doc IDs.

## 🟡 MEDIUM (UX/correctness)

### 10. No delete confirmation dialog
Hovering a document row reveals a Delete button that immediately deletes with no confirmation. Combined with bug #3 (Delete inside `<a>`), a miss-click can silently destroy a document.

### 11. Manual-only versioning with no auto-save and no version naming
The Versions panel has a single "Save Version" button and shows "No versions saved yet". No snapshots are taken automatically. Versions cannot be named. There is no "restore" button visible unless a version is saved.

### 12. Multiple panels can open simultaneously and crush the editor
The TOC panel (left) and Comments panel (right) can both be open at the same time with no coordination. When both are open, the editing area is squeezed to a narrow strip. No panel management system exists.

### 13. No floating/bubble menu on text selection
Selecting text shows no inline formatting popup. Users must scroll their eyes up to the static toolbar at the top of the page to apply formatting. All modern editors (Notion, Google Docs, Quip, etc.) show an inline bubble menu on selection.

### 14. `1 paragraphs` grammatical error in status bar
The status bar reads "1 paragraphs" when the document has a single paragraph. Should be "1 paragraph".

## 🔵 LOW / UX (polish and discoverability)

### 15. All toolbar buttons use plain text labels with no icons and no tooltips
`B  I  S  Code  H1  H2  H3  List  1.  Quote  <>  HR  Table  Image  Emoji  Find  Comment  Suggest  Page Break  TOC  Print  PDF  References  Versions  ?`

No icons. No separators between groups. No tooltips on hover. The label `<>` for Code Block is completely cryptic. The label `1.` for numbered list is ambiguous.

### 16. Toolbar button labels are inconsistent with keyboard shortcut help
- Toolbar says `Quote` → Help says `Blockquote`  
- Toolbar says `<>` → Help says `Code block`

### 17. No user account anywhere — user is always "Anonymous"
The app auto-creates an anonymous session (JWT stored in `auth_token` localStorage key, user name hardcoded as "Anonymous"). There is no login page, no profile screen, no way to set a display name. The dashboard has no user indicator, avatar, or logout button.

### 18. Dashboard shows a hard cap of 15 documents with no pagination
The API is queried without a limit, but only 15 documents are shown and there is no pagination UI, "Load more" button, or infinite scroll. Users with more than 15 documents cannot see the rest.

### 19. No document list sorting or filtering
Documents are sorted by last-updated only. There are no controls to sort by name, created date, or type, and no way to filter by document type (Document / Spreadsheet / Presentation).

### 20. No multi-select or bulk delete on dashboard
Each document can only be deleted one at a time. There is no way to select multiple documents for bulk delete, move, or export.

### 21. Copy button in Share dialog gives zero feedback
Clicking "Copy" after generating a share link does not show a "Copied!" confirmation, change the button text/colour, or display any toast. The user has no idea if the clipboard write succeeded.

### 22. Comment timestamps show only "just now"
Comments display "just now" as their timestamp. There is no absolute time shown, and no tooltip on hover with the full timestamp.

### 23. Share link URL uses `localhost:3000` in the generated link
The generated share link is `http://localhost:3000/share.html?token=...`. In a real deployment the host would be correct, but the URL is constructed from the page's `window.location.origin` in dev — and this is what would be sent to recipients. Should respect a `PUBLIC_URL` env var or similar.

### 24. No "Recently opened" or favourites section
The dashboard is a flat recency-sorted list. No recent-per-user section, no starred/favourited documents.

### 25. Delete buttons have no accessible label describing which document
All 15 Delete buttons are announced to screen readers as "Delete" with no document name context. Should be `aria-label="Delete <document name>"`.

### 26. The theme toggle icon is not self-explanatory
The icon on the theme toggle button is small and context-free. The `aria-label="Toggle theme"` is correct, but a sighted user has no visual affordance until they hover and check the label.

### 27. Permission dropdown in Share dialog is an unstyled native `<select>`
Inconsistent with the rest of the app's custom-styled components.

### 28. Spreadsheet toolbar replaces the formatting toolbar when inside a table cell — wait, correction
When inside a table cell, a second toolbar row appears with table-specific controls. The formatting row remains. However, this makes the toolbar two rows tall and visually noisy — no separators distinguish formatting actions from table actions.

### 29. Document area width is fixed narrow on wide viewports
The editor document is capped at ~560px wide, leaving large dark dead zones on both sides. There is no way to expand the editor width or toggle a focused/full-width mode.

### 30. `window.prompt()` dialog title exposes `localhost says:`
When "New Spreadsheet" or "New Presentation" is clicked, the native prompt reads `localhost says: Spreadsheet title:`. This looks like a phishing/malware alert to users unfamiliar with it.

### 31. No empty-state on dashboard for a first-time user
The dashboard immediately tries to show documents. If a real first-time user had zero documents, they would see a generic empty state — but it is not clear that the empty state is styled, and there is no onboarding guidance.

### 32. Collaborative presence indicator shows "Anonymous" for all users
When two sessions are open, "Editors: Anonymous, Anonymous" is displayed. No way to distinguish collaborators.

---

### 33. Safari (macOS): CSS and JS fail to load due to TLS error
On Safari on macOS connecting to the local dev server, all static assets fail to load:
```
[Error] Failed to load resource: A TLS error caused the secure connection to fail. (doc-list.bundle.css, line 0)
[Error] Failed to load resource: A TLS error caused the secure connection to fail. (doc-list.bundle.js, line 0)
```
The app is completely unusable in Safari. Root cause is likely the dev server serving over HTTP without a valid cert while Safari's ATS (App Transport Security) / strict TLS policy rejects the connection. The nginx config or the asset serving needs to either use a valid self-signed cert trusted by the OS keychain, or ensure assets are served over plain HTTP consistently.
