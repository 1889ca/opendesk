# Contract: kb

## Purpose

Knowledge Base entity directory for OpenDesk. Manages structured entities (people, organizations, projects, terms) with typed schemas, CRUD operations, and search. Entities are referenceable from the document editor via mention nodes.

## Inputs

- `entityCreate`: `EntityCreateInput` -- Name, subtype, and structured content fields for creating a new entity.
- `entityUpdate`: `EntityUpdateInput` -- Partial update to an existing entity's fields.
- `entityId`: `string (UUID)` -- Identifier for a specific entity.
- `workspaceId`: `string (UUID)` -- Scopes entity operations to a workspace.
- `subtype`: `EntitySubtype` -- Filters entities by subtype (person, organization, project, term).
- `query`: `string` -- Search query for filtering entities by name.

## Outputs

- `KBEntity`: `{ id, workspaceId, subtype, name, content, tags, createdBy, createdAt, updatedAt }` -- A full entity record. Content is a JSONB blob with subtype-specific structured fields.
- `KBEntitySummary`: `{ id, subtype, name }` -- Lightweight projection for mention picker and search results.

## Side Effects

- Persists entities to PostgreSQL via the `storage` module's pool.
- Deleting an entity does not cascade to document mentions (they become stale references, resolvable by ID).

## Invariants

- Entity IDs are UUIDs, generated by the caller.
- Entity names must be non-empty strings (1-200 characters).
- Subtype must be one of: `person`, `organization`, `project`, `term`.
- Content schema varies by subtype but is always a valid JSON object.
- Workspace scoping is mandatory: every query filters by `workspaceId`.
- Tags are an array of strings, defaulting to empty.

## Dependencies

- `storage` -- Provides the PostgreSQL connection pool.
- `permissions` -- Callers must check grants before invoking mutations. The module does not enforce permissions internally.

## Boundary Rules

- MUST: Scope all queries by `workspaceId`.
- MUST: Validate all inputs with Zod schemas before database operations.
- MUST: Use parameterized queries for all SQL.
- MUST: Return `null` (not throw) when a requested entity does not exist.
- MUST NOT: Import TipTap, ProseMirror, or any editor types.
- MUST NOT: Handle HTTP requests, routing, or middleware.
- MUST NOT: Enforce permissions internally.
- MUST NOT: Manage database connections or pool lifecycle.

## Verification

- Entity IDs are UUIDs -> Unit test: parse every created entity's `id` with UUID regex.
- Subtype validation -> Unit test: parse valid/invalid subtypes through schema.
- Content schema per subtype -> Unit test: validate person/org/project/term content schemas.
- Workspace scoping -> Integration test: entities in workspace A not visible from workspace B.
- Name validation -> Unit test: empty and oversized names rejected.

## MVP Scope

Implemented:
- [x] Zod schemas for KBEntity, EntityCreateInput, EntityUpdateInput
- [x] Subtype-specific content schemas (person, organization, project, term)
- [x] PostgreSQL CRUD for entities (workspace-scoped)
- [x] Search by name (ILIKE)
- [x] Filter by subtype
