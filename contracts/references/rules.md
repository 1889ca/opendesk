# Contract: references

## Purpose

Manage per-workspace bibliographic reference libraries and resolve citation queries, providing CRUD for references and tracking which documents cite which references.

## Inputs

- `referenceCreate`: `ReferenceCreateInput` -- Title, authors, type, DOI, and other bibliographic fields for creating a new reference.
- `referenceUpdate`: `ReferenceUpdateInput` -- Partial update to an existing reference's bibliographic fields.
- `referenceId`: `string (UUID)` -- Identifier for a specific reference.
- `workspaceId`: `string (UUID)` -- Scopes reference operations to a workspace.
- `documentId`: `string (UUID)` -- A document that cites a reference.
- `citationAttrs`: `CitationAttrs` -- Inline citation metadata: referenceId, optional locator, prefix, suffix.
- `importPayload`: `string` -- BibTeX/CSL-JSON string for batch import.

## Outputs

- `Reference`: `{ id, workspaceId, type, title, authors, issuedDate, containerTitle, volume, issue, pages, doi, url, isbn, abstract, publisher, language, customFields, tags, createdBy, createdAt, updatedAt }` -- A full bibliographic reference record.
- `CitationAttrs`: `{ referenceId, locator?, prefix?, suffix? }` -- Attributes stored on an inline citation mark/node in the editor.
- `BibliographyEntry`: `{ referenceId, formattedText }` -- A rendered bibliography line (future: style-dependent).
- `DocumentCitation`: `{ id, documentId, referenceId, locator?, createdAt }` -- A link between a document and a reference.

## Side Effects

- Persists references and document-citation links to PostgreSQL via the `storage` module's pool.
- Deleting a reference cascades removal of all associated document_citations rows.

## Invariants

- Reference IDs are UUIDs, generated by the caller (typically `crypto.randomUUID()`).
- DOIs are unique per workspace. Two references in the same workspace cannot share the same non-null DOI.
- All exports must round-trip: data exported as CSL-JSON and re-imported produces equivalent reference records.
- Workspace scoping is mandatory: every query filters by `workspaceId`. No cross-workspace reference leakage.
- Authors are stored as a JSON array of `{ given?, family?, literal? }` objects.
- Reference type must be one of the defined `ReferenceType` enum values.

## Dependencies

- `storage` -- Provides the PostgreSQL connection pool. References does not manage its own connection lifecycle.
- `permissions` -- Callers must check grants before invoking reference mutations. The module itself does not enforce permissions internally.
- `auth` -- Provides the `Principal` type for `createdBy` attribution. The module receives an already-resolved principal ID, never resolves identity itself.

## Boundary Rules

- MUST: Scope all queries by `workspaceId`.
- MUST: Validate all inputs with Zod schemas before database operations.
- MUST: Use parameterized queries for all SQL (no string interpolation).
- MUST: Return `null` (not throw) when a requested reference does not exist.
- MUST NOT: Import TipTap, ProseMirror, or any editor types. Citation rendering is the editor module's responsibility.
- MUST NOT: Handle HTTP requests, routing, or middleware. That is the API layer's concern.
- MUST NOT: Enforce permissions internally. Callers (API routes, middleware) are responsible for permission checks.
- MUST NOT: Manage database connections or pool lifecycle. Use the pool from the `storage` module.

## Verification

How to test each invariant:

- Reference IDs are UUIDs -> Unit test: parse every created reference's `id` with a UUID regex, assert it matches.
- DOI uniqueness per workspace -> Integration test: insert two references with the same DOI in the same workspace, assert the second insert raises a unique constraint violation. Insert the same DOI in a different workspace, assert success.
- Round-trip export/import -> Integration test (post-MVP): export references as CSL-JSON, re-import, assert deep equality on all fields.
- Workspace scoping -> Integration test: insert references in workspace A and B, list for workspace A, assert no workspace B references appear.
- Author schema validation -> Unit test: parse valid and invalid author arrays through `AuthorSchema`, assert pass/fail.
- Reference type validation -> Unit test: parse each valid type through `ReferenceTypeSchema`, assert success. Parse an invalid type, assert failure.
- No editor imports -> Code-level audit: grep module source for `tiptap`, `prosemirror`. Assert zero references.
- No HTTP concerns -> Code-level audit: grep module source for `express`, `router`, `Request`, `Response`. Assert zero references.
